Composition & delegation

Composition & delegation can be alternatives of the inheritance.

Suppose to create a class like this:

public class Adder {

  private int result = 0;

  public void add(int x) {
    result += x;
  }

  public void add(List<Integer> l) {
    for (int x : l) add(x);
  }

  public int result() {
    return result;
  }
}

Here we store the sum adding a number or a list of numbers.

Now we want also to store each value summed into the result. Initially we think to do an extension of the class above:

public class LogAdderExt extends Adder {
  private final List<Integer> seen = new ArrayList<>();

  @Override
  public void add(int x) {
    seen.add(x);
    super.add(x);
  }

  @Override
  public void add(List<Integer> l) {
    seen.addAll(l);
    super.add(l);
  }

  public List<Integer> log() {
    return List.copyOf(seen);
  }
}

Now we can create a simple client to execute the program:

public static void main(String[] args) {
  Adder a = new Adder();
  a.add(List.of(1, 2, 3));
  System.out.println(a.result());

  LogAdderExt lae = new LogAdderExt();
  lae.add(List.of(1, 2, 3));
  System.out.println(lae.result());
  System.out.println(lae.log());
}

The output will be:

6
6
[1, 2, 3, 1, 2, 3]

We can see that the sum are right while the list is duplicated.

Try to explain what is the problem: When you call the method add of the class LogAdderExt this method first things add the elements into the list. After that, it call the super.add method (so the method into the class Adder ). Now this you might think add the element and we are ok but is not. As we have override the add method this super class call the LogAdderExt implementation. This one first add the element into the list (second time) then do the result+=element. This is not the expected behavior!

Composition

Don’t extend the class Adder, use it through the composition:

public class LogAdderComp {
  private final Adder adder = new Adder();
  private final List<Integer> seen = new ArrayList<>();

  public void add(int x) {
    seen.add(x);
    adder.add(x);
  }

  public void add(List<Integer> l) {
    seen.addAll(l);
    adder.add(l);
  }

  public int result() {
    return adder.result();
  }

  public List<Integer> log() {
    return List.copyOf(seen);
  }
}

Using this method the two object are distinct and we don’t have the same problem. In fact:

public static void main(String[] args) {
  Adder a = new Adder();
  a.add(List.of(1, 2, 3));
  System.out.println(a.result());

  LogAdderComp lac = new LogAdderComp();
  lac.add(List.of(1, 2, 3));
  System.out.println(lac.result());
  System.out.println(lac.log());
}

6
6
[1, 2, 3]

!We can do something better!

Use an interface 

public interface AdderInterface {

  public void add(int x);

  public void add(List<Integer> l);

  public int result();
}

Then create a forwarder. This class implement the interface and it will forward every call method to the supertype, pass each call to the supertype.

public class AdderForwarder implements AdderInterface {

  private final AdderInterface adder;

  public AdderForwarder(AdderInterface adder) {
    this.adder = adder;
  }

  @Override
  public void add(int x) {
    adder.add(x);
  }

  @Override
  public void add(List<Integer> l) {
    adder.add(l);
  }

  @Override
  public int result() {
    return adder.result();
  }
}

Now create the new adder that extends the class:

public class LogAdder extends AdderForwarder {
  private final List<Integer> seen = new ArrayList<>();

  public LogAdder(AdderInterface adder) {
    super(adder);
  }

  @Override
  public void add(int x) {
    seen.add(x);
    super.add(x);
  }

  @Override
  public void add(List<Integer> l) {
    seen.addAll(l);
    super.add(l);
  }

  public List<Integer> log() {
    return List.copyOf(seen);
  }
}

and using this class as test all it’s all ok

public class MainEF {

  public static void use(AdderInterface a) {
    a.add(List.of(1, 2, 3));
    System.out.println(a.result());
  }

  public static void main(String[] args) {
    AdderInterface a = new Adder();
    use(a);

    LogAdder la = new LogAdder(new Adder());
    use(la);
    System.out.println(la.log());
  }
}

Another advantage is that you can provide new implementation that implements the interface, in future, and have new better implementation without change anything.
