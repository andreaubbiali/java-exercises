Default & Static methods in interfaces

Default methods enable you to add new functionality to the interfaces of your libraries and ensure binary compatibility with code written for older versions of those interfaces.

When you need to update an interface but you want that every class that implement this interface has the new method.

Example

public interface TimeClient {
    void setTime(int hour, int minute, int second);
    void setDate(int day, int month, int year);
    void setDateAndTime(int day, int month, int year,
                               int hour, int minute, int second);
    LocalDateTime getLocalDateTime();
}

The following class implements the interface

public class SimpleTimeClient implements TimeClient {
    
    private LocalDateTime dateAndTime;
    
    public SimpleTimeClient() {
        //...
    }
    
    public void setTime(int hour, int minute, int second) {
        //...
    }
    
    public void setDate(int day, int month, int year) {
        //...
    }
    
    public void setDateAndTime(int day, int month, int year,
                               int hour, int minute, int second) {
        //...
    }
    
    public LocalDateTime getLocalDateTime() {
        //...
    }
    
    public String toString() {
        //...
    }
    
    public static void main(String... args) {
        //...
    }
}

Now for example we want to add new functionality to the interface:

public interface TimeClient {
    void setTime(int hour, int minute, int second);
    void setDate(int day, int month, int year);
    void setDateAndTime(int day, int month, int year,
        int hour, int minute, int second);
    LocalDateTime getLocalDateTime();                           
    ZonedDateTime getZonedDateTime(String zoneString);
}

If you do like as above you need also to write the new method into the class SimpleTimeClient or you can add also the implementation and not only the abstract method:

package defaultmethods;
 
import java.time.*;

public interface TimeClient {
    void setTime(int hour, int minute, int second);
    void setDate(int day, int month, int year);
    void setDateAndTime(int day, int month, int year,
                               int hour, int minute, int second);
    LocalDateTime getLocalDateTime();
    
    static ZoneId getZoneId (String zoneString) {
        try {
            return ZoneId.of(zoneString);
        } catch (DateTimeException e) {
            System.err.println("Invalid time zone: " + zoneString +
                "; using default time zone instead.");
            return ZoneId.systemDefault();
        }
    }
        
    default ZonedDateTime getZonedDateTime(String zoneString) {
        return ZonedDateTime.of(getLocalDateTime(), getZoneId(zoneString));
    }
}

Extending interfaces that contain default methods:

you can do the following:

  • Not mention the default method at all, which lets your extended interface inherit the default method.
  • Redeclare the default method, which makes it abstract.
  • Redefine the default method, which overrides it.

Static methods

A static method is a method that is associated with the class in which it is defined rather than with any object. Every instance of the class shares its static methods.
